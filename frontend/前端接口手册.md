## 一、项目结构说明

```
social/
├── config/
│   └── api.js              # 前端 API 配置（Key、BaseURL）
├── services/

```

## 二、意图识别模块！！目前接的是deepseek做意图识别，是我自己花钱买的token，慎用，不要直接大规模测试

### 2.1 前端实现

文件：`services/intentService.js`

前端已封装 `recognizeIntent(userInput)` 方法，支持以下策略：

1. **优先策略**：若 `config/api.js` 中配置了 `DEEPSEEK_API_KEY`，直接调用 DeepSeek API。
2. **降级策略**：若未配置 Key 或调用失败，自动降级为本地关键词匹配。

### 2.2 DeepSeek 调用方式（烧的是我自己的token，慎用）

前端直接请求 DeepSeek 官方接口，**后续开发需要后端额外提供意图识别接口**。

```http
POST https://api.deepseek.com/chat/completions
Headers:
  Content-Type: application/json
  Authorization: Bearer {DEEPSEEK_API_KEY}

Body:
{
  "model": "deepseek-chat",
  "messages": [
    {
      "role": "system",
      "content": "你是一个意图识别助手。请分析用户输入，返回严格的JSON格式：\n{\"intent\":\"navigate|chat|unknown\",\"target\":\"GymScene|McDonaldScene|OfficeScene|null\",\"query\":\"用户原始问题或补充信息\"}\n规则：\n- 如果用户表达想去某个地方，intent为\"navigate\"，target为对应场景key。\n- 如果用户在向某个职能部门提问（如套餐、预约、教学政策），intent为\"chat\"。\n- 如果无法识别，intent为\"unknown\"。\n- query 字段保留用户原始输入。"
    },
    { "role": "user", "content": "{userInput}" }
  ],
  "response_format": { "type": "json_object" },
  "temperature": 0.3
}
```

### 2.3 意图识别输出格式

前端期望的响应结构（无论 DeepSeek 还是后端代理）：

```json
{
  "intent": "navigate",
  "target": "GymScene",
  "query": "我想去体育馆"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `intent` | string | `navigate` / `chat` / `unknown` |
| `target` | string | 目标场景 Key：`GymScene`、`McDonaldScene`、`OfficeScene`；无法识别时为 `null` |
| `query` | string | 用户原始输入或提取的核心问题 |

**意图说明**：
- `navigate`：用户想去某地，前端直接跳转对应子场景。
- `chat`：用户在提问，前端提示前往对应场景找 NPC，或由后端决定直接返回答案。
- `unknown`：无法识别，前端提示未听懂。

### 2.4 前端调用示例

```js
import { recognizeIntent } from './services/intentService.js';

const result = await recognizeIntent('我想去体育馆看看有什么活动');
// result = { intent: 'navigate', target: 'GymScene', query: '我想去体育馆看看有什么活动' }
```

## 三、智能体对话模块（NPC 后端接口预留）

### 3.1 前端预留文件

文件：`services/agentService.js`

前端已预留智能体注册表和调用接口，**需要后端提供实际的问答能力**。

### 3.2 前端智能体注册表

```js
const AGENT_REGISTRY = {
  gym_admin:  { name: '体育馆管理员', scene: 'GymScene',  role: '负责体育馆场地预约、收费、开放时间咨询' },
  mcd_manager:{ name: '麦当劳经理',   scene: 'McDonaldScene', role: '负责推荐套餐、联名活动、限定餐品信息' },
  dean_chen:  { name: '陈院长',       scene: 'OfficeScene', role: '负责本科教学政策、培养方案、学院介绍' }
};
```

### 3.3 后端需实现的接口

#### 3.3.1 智能体问答接口（必须）

```http
POST /api/agent/ask
Content-Type: application/json

Request:
{
  "agentId": "gym_admin",      // 智能体ID，见前端注册表
  "question": "今天游泳馆开吗？", // 用户问题
  "context": {                  // 可选，对话上下文
    "history": [],
    "userAicId": "AIC_xxx"
  }
}

Response:
{
  "answer": "今天游泳馆正常开放，开放时间为 8:00-22:00。",
  "agentId": "gym_admin",
  "source": "knowledge_base"   // 可选，回答来源标识
}
```

#### 3.3.2 获取智能体信息（可选）

```http
GET /api/agent/{agentId}

Response:
{
  "id": "gym_admin",
  "name": "体育馆管理员",
  "scene": "GymScene",
  "role": "负责体育馆场地预约、收费、开放时间咨询",
  "avatar": "https://..."
}
```

#### 3.3.3 获取场景下智能体列表（可选）

```http
GET /api/agents?scene=GymScene

Response:
{
  "agents": [
    { "id": "gym_admin", "name": "体育馆管理员", ... }
  ]
}
```

### 3.4 前端 NPC 对话中的智能体触发方式

前端 `SubSceneBase` 的对话数据（`dialogData.js`）支持以下行类型：

```js
// 1. 普通文本
"你好，欢迎光临！"

// 2. 用户输入框
{ input: true, placeholder: "请回答", defaultValue: "" }

// 3. 智能体回答节点（新增，预留）
{ agent: "gym_admin", question: "{userInput}" }
// 或简写：使用场景默认 agent
{ agent: true }
```

当对话流程遇到 `{ agent: ... }` 时，前端会自动调用 `askAgent()`，显示"正在思考..."，等待后端返回后展示答案。

### 3.5 前端调用示例

```js
import { askAgent, getAgentInfo, getAgentsByScene } from './services/agentService.js';

// 问答
const res = await askAgent('gym_admin', '最近有什么优惠套餐？');
console.log(res.answer);

// 获取信息
const info = await getAgentInfo('gym_admin');
const agents = await getAgentsByScene('GymScene');
```

## 四、好友关系模块（斯坦福小镇式社交系统）

### 4.1 前端预留文件

文件：`services/socialService.js`

当前所有函数返回 Mock 数据，**需要后端实现实际接口**。

### 4.2 用户身份标识（AIC 号）

- 字段名：`aicId`
- 说明：全局唯一身份标识，相当于用户 ID。前端在调用社交接口时，均通过 `aicId` 标识用户。
- 获取方式：由后端认证服务发放，前端存储在 `localStorage` 或内存中。

### 4.3 后端需实现的接口

#### 4.3.1 获取用户档案

```http
GET /api/user/{aicId}

Response:
{
  "success": true,
  "data": {
    "aicId": "AIC_001",
    "name": "张三",
    "avatar": "https://...",
    "friends": ["AIC_002", "AIC_003"]
  }
}
```

#### 4.3.2 申请添加好友

```http
POST /api/social/friend-request
Content-Type: application/json

Request:
{
  "fromAicId": "AIC_001",
  "toAicId": "AIC_002"
}

Response:
{
  "success": true,
  "message": "好友申请已发送"
}
```

#### 4.3.3 获取好友列表

```http
GET /api/social/friends?aicId=AIC_001

Response:
{
  "success": true,
  "data": [
    { "aicId": "AIC_002", "name": "李四", "avatar": "https://..." }
  ]
}
```

#### 4.3.4 获取历史对话

```http
GET /api/social/chat-history?userA=AIC_001&userB=AIC_002

Response:
{
  "success": true,
  "data": [
    {
      "from": "AIC_001",
      "to": "AIC_002",
      "message": "你好！",
      "timestamp": "2026-05-06T10:30:00Z"
    }
  ]
}
```

#### 4.3.5 保存对话消息

```http
POST /api/social/chat-message
Content-Type: application/json

Request:
{
  "from": "AIC_001",
  "to": "AIC_002",
  "message": "明天一起去体育馆吗？"
}

Response:
{
  "success": true
}
```

### 4.4 前端调用示例

```js
import {
  getUserProfile,
  addFriend,
  getFriendList,
  getChatHistory,
  saveChatMessage
} from './services/socialService.js';

const profile = await getUserProfile('AIC_001');
await addFriend('AIC_001', 'AIC_002');
const friends = await getFriendList('AIC_001');
const history = await getChatHistory('AIC_001', 'AIC_002');
await saveChatMessage('AIC_001', 'AIC_002', '你好！');
```

## 五、配置说明（前端侧）

文件：`config/api.js`

```js
export const API_CONFIG = {
  // DeepSeek 意图识别（前端直接调用，也可走后端代理）
  DEEPSEEK_API_KEY: '',         // <-- 用户手动填入
  DEEPSEEK_BASE_URL: 'https://api.deepseek.com',
  DEEPSEEK_MODEL: 'deepseek-chat',

  // 后端服务基地址（智能体、好友关系等）
  BACKEND_BASE_URL: '',         // <-- 对接时填入后端域名

  // 调试：true 时强制走本地 fallback，不走远程
  DEBUG_FORCE_FALLBACK: false
};
```

**注意**：
- `DEEPSEEK_API_KEY` 由我提供，后续要将 `DEEPSEEK_BASE_URL` 改为后端代理地址，重新做意图识别。
